CHAPTER 01

What is TouchDesigner?

TouchDesigner is a visual programming environment made by Derivative. Instead of writing traditional code line-by-line, you connect nodes (called Operators) on a canvas. Data flows between them in real time — making it ideal for live performances, interactive installations, and in Sentio's case, projecting EEG-driven generative visuals onto physical garments.

The Node-Based Canvas

Every TouchDesigner project lives inside a Network Editor — a zoomable canvas where you place and wire Operators. Think of it like a visual patch bay: each Operator does one job (generate an image, read a sensor, apply a filter) and passes its result to the next node downstream.

💡
Tip

Press Tab on an empty area of the canvas to open the Operator browser. Type the first letters of any Operator to filter the list instantly. Press P to open the parameters panel for the selected node.

Operator Families

Operators are grouped into five families, each with its own wire colour and data type:

TOP
Texture Operator
2D images & video frames. Purple wires. Used for all visual output in Sentio.
CHOP
Channel Operator
Time-series numbers & signals. Green wires. Carries all EEG band values.
DAT
Data Operator
Text, tables & scripts. Teal wires. Python callbacks live here.
SOP
Surface Operator
3D geometry & particles. Blue wires. Used for the spark particle layer.
COMP
Component
Containers, UI & output. Grey wires. Stoner projection mapper lives here.
⚠️
Important

You cannot wire a CHOP directly into a TOP input — they carry different data types. Reference CHOP values inside TOP parameters using the expression syntax op('chop_name')['channel_name'] instead of a physical wire.

CHAPTER 02

Project Setup & First Launch

Installing TouchDesigner

TouchDesigner is free for non-commercial use. Download the installer from derivative.ca/download. During installation create a free account — this activates the non-commercial licence automatically on first launch.

1
Download the installer
Go to derivative.ca/download and grab the latest stable build for your OS (Windows recommended for Sentio).
2
Run the installer
Follow the on-screen steps — default options are fine. TouchDesigner installs to C:\Program Files\Derivative\TouchDesigner.
3
Log in with your Derivative account
On first launch TouchDesigner asks for your credentials. Sign in to activate the licence. The title bar will confirm Non-Commercial in green.
ℹ️
Non-Commercial Licence

The free licence limits output resolution to 1280 × 1280 px. For projecting onto a mannequin in a studio context this is fully sufficient. A paid licence (from $600 USD) removes this restriction.

💡
TouchDesigner 2025 — What's New for Sentio

The latest stable build is 2025.32460 (March 2026). Key changes that affect this project:

  • Python 3.11.10 — all scripts in scripts/ are fully compatible. No changes required.
  • Point Operators (POPs) — a new GPU-based operator family for instanced geometry. Sentio uses the legacy Particle SOP which still works, but POPs are the recommended upgrade path for higher instance counts. See Chapter 5 for details.
  • Render Simple TOP (new) — renders geometry without needing a Camera COMP or Light COMP. Simplifies the particle render chain; see Chapter 9.
  • Layer Mix TOP (new) — an alternative to Composite TOP that handles unlimited layers with per-layer controls. Either works for Sentio's compositing step.
  • Python Environment Manager (new) — at Edit → Preferences → Python you can now create virtual environments inside TD directly. Useful if you install additional Python packages (e.g. python-osc).
  • TDI Library (new) — install the tdi package for full VS Code autocomplete and inline help when editing scripts outside TouchDesigner.
  • Minimum GPU memory: 4 GB — 8 GB recommended if using POPs. Verify your machine meets this before starting.
  • macOS users — macOS 13 Ventura or higher is required for 2025 builds.

Create all five top-level containers at the root level first, then double-click into each one to build its contents. Work in this order: osc → signal → visual → projection → debug. Each container's output is used by the next, so building left-to-right means you always have a valid input to wire when you need it.

Container 1 — container_osc

Receives all incoming OSC messages from the Python backend and fires a Python callback each time a new packet arrives. Nothing else in the network should touch the raw OSCIn CHOP directly — always reference container_osc/null_osc_out from other containers.

1
Double-click container_osc to enter it
Breadcrumb should read /project1/container_osc. You are now inside an empty network — all operators you place here live inside this container.
2
Add an OSCIn CHOP — rename it oscin1
Tab → type OSC In → double-click OSCIn CHOP.
Open parameters (P) and set:
• Port → 7000
• Address Scope → /sentio/*
• Active → On
You should immediately see channels appear in the node viewer once the Python backend is running.
3
Add a CHOP to DAT — rename it chop_to_dat1
Tab → type CHOP to DAT → place it to the right of oscin1.
Wire: oscin1 output → chop_to_dat1 input.
Open parameters → DAT Type: Channel per Row. This gives a live table of channel names and values for debugging.
4
Add a DAT Execute — rename it execute_osc
Tab → type DAT Execute → place it below oscin1.
Open parameters (P) and set each field exactly as follows:
• DATs → type oscin1 (or drag the node into the field). This tells the DAT Execute which table to watch.
• Table Change → Off — fires when the table structure changes (columns added/removed); not needed for OSC.
• Row Change → On — fires each time a new OSC message arrives and updates a row. This is the correct trigger for the emotion callback.
• Column Change → Off
• Execute → Off — critical: leave this Off. If it is set to Start of Frame the script runs 60× per second regardless of incoming data, wasting CPU.

Your parameters panel should look like this when correctly set:
Active On DATs: oscin1 Row Change: On Table Change: Off Execute: Off

Then click Edit… to open the script editor and paste the callback below.
★
Paste this script into execute_osc
Click Edit… on the DAT Execute, select all existing text, and replace with:
Python — execute_osc (DAT Execute · TD 2025 signature) 
# TD 2025 onReceiveOSC signature
# Fires on Row Change — once per incoming OSC packet
def onReceiveOSC(dat, rowIndex, message, bytes, timeStamp, address, args, peer):
    emotion = str(op('oscin1')['emotion'])

    palettes = {
        'calm':     {'hue': 200, 'flow': 0.15, 'dist': 0.10},
        'focused':  {'hue': 42,  'flow': 0.55, 'dist': 0.30},
        'stressed': {'hue': 0,   'flow': 0.88, 'dist': 0.70},
        'relaxed':  {'hue': 175, 'flow': 0.12, 'dist': 0.06},
        'excited':  {'hue': 285, 'flow': 0.75, 'dist': 0.55},
    }

    p = palettes.get(emotion, palettes['calm'])

    # Write to the Constant CHOPs in container_signal
    op('../container_signal/constant_hue').par.value0  = p['hue']
    op('../container_signal/constant_flow').par.value0 = p['flow']
    op('../container_signal/constant_dist').par.value0 = p['dist']
🚨
Common mistake — Execute set to "Start of Frame"

When you first place a DAT Execute, the Execute parameter defaults to Start of Frame. This causes the callback to run every single frame (60×/sec), meaning it constantly overwrites the Constant CHOPs even when no OSC data has arrived. Always set Execute → Off and Row Change → On so the script only fires when a real OSC packet is received.

5
Add a Null CHOP — rename it null_osc_out
Tab → type Null → place it to the far right of the network.
Wire: oscin1 output → null_osc_out input.
This is the output reference for this container — all other containers wire from here, not from oscin1 directly.
6
Press U to go back to the root network
You should see container_osc in the root. Its viewer will show the Null CHOP output — a small waveform or number strip — confirming data is flowing.

Container 2 — container_signal

Smooths, scales, and maps the raw EEG band values into visual parameters. The emotion mapping script writes to three Constant CHOPs here; TOP expressions in container_visual read from them every frame.

1
Double-click container_signal to enter it
Breadcrumb: /project1/container_signal. You are in an empty network.
2
Add a Null CHOP — rename it null_osc_in
Tab → Null → place on the far left.
This is the input gate. To wire it from outside: press U to return to root, then drag a wire from container_osc/null_osc_out into container_signal's input port. Re-enter the container — null_osc_in will now show live EEG channels.
3
Add a Filter CHOP — rename it filter1
Tab → Filter → place to the right of null_osc_in. Wire: null_osc_in → filter1.
Parameters: Filter Type → Gaussian · Filter Width → 0.3 (seconds) · Extend Left/Right → Hold.
4
Add a Math CHOP — rename it math1
Tab → Math → place after filter1. Wire: filter1 → math1.
Parameters: open the Range tab → From 0 to 1, To 0 to 1. This clamps any values that stray outside the expected 0–1 band-power range.
5
Add a Null CHOP — rename it null_bands
Wire: math1 → null_bands. This is Output A — all visual operators reference smoothed EEG bands from here using op('container_signal/null_bands')['alpha'] etc.
6
Add three Constant CHOPs — name them constant_hue, constant_flow, constant_dist
Tab → Constant → place three below the main chain, side by side.
For each: open parameters → Channel 0 Name → type hue / flow / dist.
Default values: constant_hue → 200 · constant_flow → 0.15 · constant_dist → 0.10.
These are written to in real time by the DAT Execute callback when an emotion packet arrives.
7
Add a Merge CHOP, then a Null CHOP — rename the Null null_params
Tab → Merge → wire all three Constant CHOPs into the Merge CHOP inputs.
Then wire: Merge → null_params. This is Output B — exposes hue, flow, and dist as a single CHOP for convenience.
8
Press U to return to the root
The container_signal viewer should now show the smoothed band channels. If it shows nothing, confirm the wire from container_osc/null_osc_out is connected to container_signal's input.

Container 3 — container_visual

The visual engine. Builds the instanced capsule geometry (Polyhop-style) driven by the noise field, composites it with a feedback trail, colour-grades by emotion, and outputs a single texture to container_projection.

The container holds two alternative geometry pipelines: Option A uses a Particle SOP + Geometry COMP (classic, compatible with all TD versions); Option B uses a POP Network COMP + Render Simple TOP (TD 2025 GPU instancing — recommended). Build one; leave the other commented out or disabled.

1
Double-click container_visual to enter it
Breadcrumb: /project1/container_visual. Note: this container does not have a CHOP input — it reads from container_signal using expressions, not wires.
2
Add a Noise TOP — rename it noise1
Tab → Noise → place on the left.
Parameters:
• Type → Sparse
• Resolution → 1280 × 720 (or your projector native)
• Roughness → click the expression toggle (= icon) → type op('container_signal/null_bands')['beta']
• Translate Z → expression: absTime.seconds * op('container_signal/constant_flow').par.value0
3
Add Remap TOP → Blur TOP — rename remap1, blur1
Wire: noise1 → remap1 → blur1.
Blur TOP parameters: Size → 6. Leave Remap at defaults initially; adjust the curve once you see the visual.
4a
Option B (TD 2025) — Add a POP Network COMP — rename it popnet1
Tab → POP Network → place below blur1.
Double-click popnet1 to enter the POP network. Inside, add:
• POP Source (rename popsource1): Birth Rate → 0, Constant Birth Rate → 450
• POP Noise (rename popnoise1): wire from popsource1. Amplitude expression → op('container_signal/null_bands')['alpha'] * 0.4 + 0.05
• POP Color (rename popcolor1): wire from popnoise1. Set Color Type → Ramp
• Capsule SOP (rename capsule_shape): Radius 0.02, Height 0.12 — this is the instance geometry shape
Press U to exit back to container_visual.
4b
Option B — Add a Render Simple TOP — rename it renderSimple1
Tab → Render Simple → place to the right of popnet1.
Parameters:
• Geometry → drag popnet1 into this field (or type its path)
• Resolution → match noise1 resolution (1280 × 720)
• Background Color → 0, 0, 0, 0 (fully transparent — capsules render over black, composited via Screen blend)
5
Add a Composite TOP — rename it composite1
Tab → Composite → place centrally.
• Input 0 (first wire) → renderSimple1 output (the fresh geometry frame)
• Input 1 (second wire) → leave unconnected for now — you'll close the feedback loop in the next step
• Parameters: Operation → Screen
6
Add a Feedback TOP + Level TOP to create the trail loop
Tab → Feedback → rename feedback1. Parameters: Target TOP → type composite1.
Wire: feedback1 output → composite1 Input 1 (closing the loop).

Add Level TOP → rename level_fade. Wire: feedback1 output → level_fade → back into feedback1 input. Parameters: Brightness → 0.985. This fades old frames slightly each tick so trails don't accumulate to white.
7
Add Level TOP → HSV Adjust TOP → Bloom TOP — the colour grade chain
Wire: composite1 → level1 → hsvadjust1 → bloom1.

Level TOP (level1) → Brightness expression: op('container_signal/null_bands')['delta'] * 0.6 + 0.3
HSV Adjust TOP (hsvadjust1) → Hue Offset expression: op('container_signal/constant_hue').par.value0 / 360.0 · Saturation expression: op('container_signal/null_bands')['gamma'] * 0.5 + 0.6
Bloom TOP (bloom1) → Threshold 0.55 · Blur Radius 12 · Blend 0.45
8
Add an Out TOP — rename it out1
Wire: bloom1 → out1. This is the official output of the container. Press U — you should see the full visual rendering inside container_visual's thumbnail in the root network.

CHAPTER 04

Receiving Data via OSC

What is OSC?

OSC (Open Sound Control) is a lightweight network protocol designed for real-time communication between creative software tools. The Sentio Python backend sends EEG band values and emotion data as OSC messages to TouchDesigner over the local loopback network. Each message has a path (like a URL address) and one or more numeric or string values.

OSC message /sentio/alpha → value: 0.72 float 0–1
OSC message /sentio/emotion → value: "calm" string

Adding the OSCin CHOP

Follow these steps inside TouchDesigner to start receiving the Sentio data stream:

1
Open the Operator browser
Press Tab in the empty Network Editor canvas.
2
Add the OSCin CHOP
Type OSC in the search box and double-click OSCin CHOP. A green node appears on the canvas.
3
Open parameters (press P)
With the OSCin node selected, press P to open the Parameters panel on the right side.
4
Set Network Port to 7000
This matches the port configured in the Sentio backend's OSC_PORT environment variable.
5
Set Address Pattern to /sentio
This filters messages — only paths starting with /sentio will be received.
6
Click Active to enable the node
The LED indicator turns green when data is arriving. You should see waveform activity in the node preview.
⚠️
Before you activate

Make sure the Python backend is running first: cd backend && uvicorn main:app --reload Then start a session in the React frontend to trigger the EEG stream. Without an active session the OSCin CHOP will receive no data even if the backend is running.

CHAPTER 05

Building the Processing Network

Raw OSC values are noisy and update at high frequency. Before routing them to the visual operators, build this CHOP processing chain to smooth and scale the signals.

CHOP CHAIN
OSCin CHOP → Filter CHOP → Math CHOP → Null CHOP reference point for TOPs
OSCin CHOP → DAT Execute → Constant CHOP × 3 hue / flow / distortion

Smoothing with Filter CHOP

A Filter CHOP applies a low-pass filter to every incoming channel, removing rapid jitter so visual changes feel fluid rather than twitchy.

ParameterValueWhy
Filter TypeGaussianSmooth bell-curve response, no ringing artefacts
Filter Width0.3 secSmooths ~0.3 s of history; increase for slower visuals
Extend Left / RightHoldPrevents edge artefacts at start/end of stream

Mapping Emotion to Visual Parameters

The emotion label is a string and needs converting to numeric values. Paste this script into a DAT Execute node — it fires every time a new OSC message arrives:

Python — DAT Execute 
# TD 2025 signature — address and args are now passed directly
# Old signature (pre-2022):  onReceiveOSC(dat, message, bytes, timeStamp)
def onReceiveOSC(dat, rowIndex, message, bytes, timeStamp, address, args, peer):
    emotion = str(op('oscin1')['emotion'])

    palettes = {
        'calm':     {'hue': 200, 'flow': 0.15, 'dist': 0.10},
        'focused':  {'hue': 42,  'flow': 0.55, 'dist': 0.30},
        'stressed': {'hue': 0,   'flow': 0.88, 'dist': 0.70},
        'relaxed':  {'hue': 175, 'flow': 0.12, 'dist': 0.06},
        'excited':  {'hue': 285, 'flow': 0.75, 'dist': 0.55},
    }

    p = palettes.get(emotion, palettes['calm'])

    # Write to Constant CHOPs — referenced by TOP parameters
    op('constant_hue').par.value0  = p['hue']
    op('constant_flow').par.value0 = p['flow']
    op('constant_dist').par.value0 = p['dist']
⚠️
TD 2025 — Updated OSC callback signature

TouchDesigner 2022 and later changed the onReceiveOSC callback to include three extra parameters: address, args, and peer. Using the old four-parameter signature still runs without an error, but you cannot access the address or argument values directly — you must read them from the CHOP row instead. The version above uses the correct 2025 signature and gives you all OSC data inline.

This chapter builds the full visual pipeline — an instanced geometry system inspired by Simon Alexander-Adams (Polyhop) in which hundreds of capsule/pill forms drift, rotate and recolour in response to EEG data. Every parameter reacts in real time to incoming EEG data.

Feedback Loop for Particle Trails

A Feedback TOP blends each new frame with the previous one, creating the long glowing trails characteristic of the Anadol aesthetic.

TouchDesigner — Feedback TOP settings 
# Feedback TOP — wire after the Composite output
Target TOP:   drag your composite output node here

💡
Wire order matters

Connect the Feedback TOP output back as the second input of the Composite TOP (not the first). The first input should be the fresh frame from the Noise/Remap chain. If wired backwards the feedback will dominate and the visual will lock up on the first frame.

Instance Layer — Foreground Geometry

The foreground layer replicates the Polyhop instancing aesthetic: hundreds of capsule instances noise-displaced in position and rotation, rendered with additive blending over the noise field. Two approaches are available depending on your TD version and GPU.

💡
Which approach to use in TD 2025?

Particle SOP + Render TOP — works on all TD versions, requires a Camera COMP and Light COMP in the scene.

Point Operators (POPs) + Render Simple TOP — new in TD 2025. GPU-based, no Camera or Light needed. Handles tens of thousands of instances at full frame rate. Use this if your GPU has ≥ 8 GB VRAM.

Option A — Particle SOP (all versions)

4

Projection Mapping onto the Mannequin

Projection mapping makes a flat projector image conform precisely to the surface of a 3D object. In Sentio we project the fluid visual onto a headless dress-form mannequin — a standard dressmaker's torso on a pole stand — turning its white surface into a living EEG canvas that pulses in real time with the wearer's emotional state.

Physical Setup Checklist

  • Mannequin type: Use a headless torso dress form — no arms, no head, mounted on a centre pole stand. The flat, matte-white surface of a professional dressmaker's form provides the ideal projection canvas. Fabric-padded foam bodies (e.g. standard dressmaking mannequins) work well; avoid hard plastic with a shiny finish.
  • Projector position: Mount directly in front of and slightly above the mannequin. Lens axis should be roughly perpendicular to the torso front face. Ideal throw distance: 1.5 – 2.5 m. Adjust keystone in the projector's on-screen menu if the image tapers at top or bottom.
  • Room lighting: Darken the room as much as possible. Ambient light washes out projected colour. Black-out curtains or a dark exhibition booth work best.
  • Mannequin surface: White or very light grey matte surfaces reflect projected colour most faithfully. Avoid glossy or sequined fabric — they create hotspot reflections and colour shifts.
  • Stand stability: The mannequin must not move during or after calibration. Clamp the pole base to the floor with gaffer tape or use a heavy tripod-style weighted base. Even 2–3 cm of movement will misalign the projection.
  • Projection area: Frame the projector output to cover from the shoulder line to the hip/waist hem of the dress form — roughly 40–60 cm tall on the physical surface. Crop using the Stoner COMP control points (see Step 5 below) rather than changing the projector zoom.
  • Projector resolution: Set to the projector's native resolution (e.g. 1920 × 1080). Match this value in the TouchDesigner Window COMP settings.
  • Laptop placement: Keep the laptop running Sentio within HDMI cable reach. A 3–5 m cable avoids clutter near the projection area.

Single-channel numeric values written by Python, read by TOP expressions.

Manual Wiring — TOP Operators

noise1
Noise TOP
param_handler.py
The base curl flow field. param_handler writes period, amp, and tz every frame.
  • Add TOP → Noise TOP. Name it noise1.
  • Parameters: Type = Sparse, Monochrome = Off, Resolution = 1920×1080.
  • Set Translate Z to expression mode: absTime.seconds * op('constant_flow')['flow']
  • param_handler.py sets noise1.par.period and noise1.par.amp each frame.
  • Wire noise1 → remap1.
blur1
Blur TOP
param_handler.py
Edge softening. Blur radius driven by alpha band (calm = softer, focused = sharper).
  • Add TOP → Blur TOP. Name it blur1. Wire remap1 → blur1.
  • Set Size to expression: float(op('sentio_params')['preset_blur_radius', 1])
  • Or leave it unset — param_handler.py writes it directly via blur1.par.size.
feedback1
Feedback TOP
param_handler.py
Trail persistence loop. High theta = very long trails. Critical wiring — must loop back.
  • Add TOP → Feedback TOP. Name it feedback1.
  • Set Target TOP parameter to composite1 (the node it feeds back into).
  • Wire feedback1 → composite1 as input 1 (second input slot).
  • Wire blur1 → composite1 as input 0 (first input slot).
  • param_handler.py writes feedback1.par.feedback each frame.
hsvAdjust1
HSV Adjust TOP
emotion_mapper.py
color_palette_manager.py
Emotion colour. Hue driven by emotion preset, saturation by palette + confidence.
  • Add TOP → HSV Adjust TOP. Name it hsvAdjust1. Wire level1 → hsvAdjust1.
  • Set Hue Shift expression: op('constant_hue')['hue'] / 360.0
  • Set Saturation Mult expression: float(op('sentio_params')['preset_saturation', 1]) + 0.3
  • Both emotion_mapper.py and color_palette_manager.py write to constant_hue and sentio_params to drive this.
bloom1
Bloom TOP
param_handler.py
Luminous glow. Gamma band lowers the threshold — peak focus creates more bloom.
  • Add TOP → Bloom TOP. Name it bloom1. Wire hsvAdjust1 → bloom1.
  • Set Threshold expression: float(op('sentio_params')['preset_bloom_threshold', 1])
  • Set Size expression: op('null_chop')['gamma'] * 0.5 + 0.2
  • Wire bloom1 → out1.
render1
Render TOP
particle_controller.py
Renders the instance/particle layer as a transparent TOP to composite over the fluid visual.
  • TD 2025 (recommended): Use Render Simple TOP instead — it requires no Camera or Light COMP. Set Geometry to your POP Network COMP or Geometry COMP directly.
  • All versions: Add TOP → Render TOP. Name it render1. Set background RGBA to all 0 (transparent). Assign Camera = cam1 and Lights = light1.
  • Wire render1 → composite1 as input 2 (third slot, Screen blending).
  • particle_controller.py sets particles1.par.birthrate, life, turbulence etc. each frame.

Frame Execution Order

The execDat Execute DAT calls every module script in this exact order on each frame. The order matters — each script depends on data written by the script before it.

1
signal_processor.py
onFrameStart(frame)
Reads raw bands from sentio_params → applies smoothing → writes proc_* values and pushes to constant_flow / constant_dist CHOPs.
2
signal_quality_monitor.py
onFrameStart(frame)
Evaluates rolling quality window → classifies state → enters or exits fallback mode → writes mode to sentio_params.
3
emotion_mapper.py
tick_transition(frame)
Advances the emotion preset blend → writes all preset_* rows to sentio_params → updates constant_hue CHOP.
4
color_palette_manager.py
tick_palette_transition(frame)
Advances palette blend → extracts dominant hue → updates hsvAdjust1.par.hueshift and hsvAdjust1.par.satmult.
5
param_handler.py
onFrameStart(frame)
Reads preset_* + proc_* rows → applies per-band nonlinear curves → smooth-writes to noise1, blur1, feedback1, level1, hsvAdjust1, bloom1.
6
particle_controller.py
onFrameStart(frame)
Reads bands + preset_particle_rate → maps to birth rate, lifespan, turbulence, drag → writes to particles1, geo1, render1.
7
debug_utils.py (optional)
tick_watches(frame)
Prints any registered operator parameter watches to the Textport. Only active when watches are registered manually.
ℹ️
OSC callbacks run separately

osc_callbacks.py runs outside this frame loop — it fires as a DAT Execute callback whenever a new OSC packet arrives from the backend (up to ~60 times per second). It writes raw values to sentio_params which the frame loop then picks up on the next frame.
