CH / 01

Overview

The Sentio LED T-shirt integration puts the EEG emotion pipeline directly onto a wearable garment. An ESP32 microcontroller connects to your home WiFi network and opens a persistent WebSocket connection to the same Python backend that drives the browser frontend. It receives emotion and EEG band power data in real time and renders one of four generative patterns onto a 16×16 WS2812B LED matrix sewn into the chest panel of a t-shirt.

There is no PC required between the headband and the shirt — the ESP32 handles all animation logic locally at 30 FPS. The backend simply streams JSON frames containing emotion labels and band powers; the microcontroller decides which pattern to draw and how bright to make it based on the signal_quality field.

Key design decisions

Decision Details
No TouchDesigner required All pattern rendering runs on the ESP32. The browser frontend and the ESP32 shirt can operate simultaneously from the same backend stream.
Fully wireless ESP32 connects over standard 2.4 GHz WiFi. No USB tether needed during performance.
Same backend endpoint Uses /ws/brain-stream — identical to the frontend. No changes to the Python backend are needed.
4 patterns, 5 emotions calm/relaxed → Ondas Fluidas, focused → Padrão Geométrico, stressed/excited → Pulsos Rítmicos, neutral → Estrelas e Partículas.
Signal quality → brightness signal_quality (0–100) scales overall LED brightness. Below threshold the idle breathing animation plays.
Heartbeat filtering Backend heartbeat frames (type: "heartbeat") are detected and silently discarded before JSON parsing.
Idle animation A gentle blue breathing pulse plays when no valid EEG signal is present, so the shirt is never dark while worn.
ℹ
Info

All four patterns are rendered at a fixed 30 FPS target using FastLED.delay(). On the ESP32-WROOM-32 the main loop runs well under the 33 ms frame budget even for the most complex pattern (Estrelas e Partículas with 48 independent stars).

Wiring diagram

ESP32 GPIO 5  ──[330Ω]──→  WS2812B DIN
ESP32 GND     ────────────  WS2812B GND
5V supply     ────────────  WS2812B 5V

⚠
Danger

Never power the matrix from the ESP32's 3.3 V pin. The on-board regulator cannot supply the current and will overheat. Always use a dedicated 5 V rail rated at least 3 A.

⚠
Warning

At MAX_BRIGHTNESS = 255, 256 LEDs all showing white can draw up to ~15 A peak. Keep MAX_BRIGHTNESS at 160 or below when running on battery — at that level peak draw is approximately 4.9 A, which most 5 V USB power banks can handle.

Power supply sizing

Brightness Colour scenario Est. current
160 Single colour (e.g. solid cyan) ~2.5 A
160 Multi-colour mixed pattern ~3.5 A
255 Full white — all LEDs 100% ~15 A

Software Setup

You need the Arduino IDE, the ESP32 board support package, and three libraries. Follow these steps in order.

1
Install Arduino IDE 2.x
Download and install the latest Arduino IDE 2 from https://www.arduino.cc/en/software. Version 2 is required for the improved Library Manager.
2
Add ESP32 board support
Open File → Preferences, paste the Espressif boards URL into Additional Boards Manager URLs, then open Tools → Board → Boards Manager, search for esp32, and install the package by Espressif Systems.
3
Install FastLED library
Open Tools → Manage Libraries, search FastLED, and install the library by Daniel Garcia.
4
Install arduinoWebSockets library
In Library Manager search arduinoWebSockets and install the library by Markus Sattler (version 2.x).
5
Install ArduinoJson v6.x
Search ArduinoJson and install the library by Benoit Blanchon. Make sure you install version 6.x, not version 7 — the API differs.
6
Select board
Go to Tools → Board → ESP32 Arduino → ESP32 Dev Module. Leave all other settings at their defaults.
7
Select port
Connect the ESP32 via USB, then choose the correct device under Tools → Port. On Windows it will appear as a COM port; on macOS/Linux as /dev/tty.usbserial-*.

config.h

All user-editable settings live in a single config.h header. Edit this file before uploading to the board — you must at minimum set your WiFi credentials and the backend host IP.

C++ — config.h 
// ─── LED matrix ─────────────────────────────────────────
#define LED_PIN           5     // GPIO pin → WS2812B DIN
#define MATRIX_W         16    // columns
#define MATRIX_H         16    // rows
#define NUM_LEDS         (MATRIX_W * MATRIX_H)
#define MATRIX_FLIP_Y    true  // flip if patterns appear upside-down
#define MATRIX_SERPENTINE true // odd rows run right-to-left

#define MAX_BRIGHTNESS   160   // 0–255; keep ≤180 on battery
#define COLOR_ORDER      GRB   // WS2812B = GRB (some clones differ)
#define LED_TYPE         WS2812B

// ─── WiFi ────────────────────────────────────────────────
#define WIFI_SSID        "YOUR_WIFI_SSID"
#define WIFI_PASSWORD    "YOUR_WIFI_PASSWORD"

// ─── Backend WebSocket ───────────────────────────────────
#define WS_HOST          "192.168.1.100"  // backend machine IP
#define WS_PORT          8000
#define WS_PATH          "/ws/brain-stream"
#define WS_RECONNECT_MS  3000             // ms between reconnect attempts

// ─── Animation ──────────────────────────────────────────
#define TARGET_FPS       30
#define SIGNAL_THRESHOLD 20   // below this = idle pattern regardless of emotion
💡
Tip

Find the backend machine's IP address with ipconfig on Windows or ip a on Linux/macOS. Look for the address on your local network interface (usually 192.168.x.x or 10.0.x.x). Make sure the backend machine and the ESP32 are on the same WiFi network.

LED Matrix Addressing

A 16×16 WS2812B flexible matrix uses serpentine wiring — a single continuous strip snakes back and forth through the panel. Even rows run left to right; odd rows run right to left. Without compensating for this in software, diagonal and symmetric patterns would appear distorted.

Serpentine layout

LED index layout (rows 0–2 shown)
Row 0:  00 → 01 → 02 → 03 → ... → 15   (left to right)
Row 1:  31 ← 30 ← 29 ← 28 ← ... ← 16   (right to left)
Row 2:  32 → 33 → 34 → 35 → ... → 47   (left to right)
  ...

The xy() helper function

All patterns call this helper to convert an (x, y) coordinate into the correct LED index in the linear leds[] array.

C++ 
uint16_t xy(uint8_t x, uint8_t y) {
  // Flip Y axis if matrix is installed with LED 0 at the bottom
#if MATRIX_FLIP_Y
  y = (MATRIX_H - 1) - y;
#endif
#if MATRIX_SERPENTINE
  if (y & 1) {
    // Odd row — reverse direction
    return (uint16_t)y * MATRIX_W + (MATRIX_W - 1 - x);
  }
#endif
  return (uint16_t)y * MATRIX_W + x;
}
💡
Tip — MATRIX_FLIP_Y

If your flexible matrix has LED index 0 at the bottom-left corner (common for strips wired bottom-to-top), set MATRIX_FLIP_Y true in config.h. Patterns will then appear right-side-up when the shirt is worn. If patterns appear correct but mirrored horizontally, the serpentine direction of your specific panel may differ — toggle MATRIX_SERPENTINE.

Emotion → Pattern Map

The ESP32 reads the emotion string from each WebSocket frame and selects a pattern function and base hue. The key EEG band powers modulate speed, scale, and density within the pattern.

Emotion Pattern Base hue Key EEG driver
calm Ondas Fluidas Cyan-blue 210° alpha
relaxed Ondas Fluidas Cyan-blue 210° alpha + theta
focused Padrão Geométrico Amber 40° beta
stressed Pulsos Rítmicos Red 0° beta
excited Pulsos Rítmicos Magenta 285° beta + gamma
neutral Estrelas e Partículas Magenta 285° alpha
no signal Idle breathing Blue —
ℹ
Info

The signal_quality value (0–100) from the backend directly scales the overall LED brightness via FastLED.setBrightness(). When signal_quality drops below SIGNAL_THRESHOLD (default 20), the emotion field is ignored and the idle breathing pattern plays regardless of what emotion the classifier reports.

Ondas Fluidas

Ondas Fluidas is a plasma sine-wave overlay. Three overlapping sine waves at different spatial and temporal frequencies create a smooth, continuously flowing colour field — like light passing through rippling water. It is used for the calm and relaxed emotional states.

🌊
Ondas Fluidas
calm · relaxed — hue 210° (cyan-blue)
Wave speed proportional to beta — faster when the user becomes more focused or stressed
Wave scale proportional to alpha — wider, gentler waves when deeply relaxed
Base hue 140 in FastLED HSV — maps to a saturated cyan-blue

Core render loop

C++ — pattern_ondas.cpp 
// nx, ny are normalised coords in [0,1] for each pixel
float speed = 0.6f + state.beta  * 3.0f;
float scale = 0.7f + state.alpha * 1.4f;

float w1 = sinf(nx * scale * 6.28f + t * 0.001f  * speed);
float w2 = sinf(ny * scale * 6.28f + t * 0.0008f * speed);
float w3 = sinf((nx + ny) * scale * 4.5f + t * 0.0006f * speed);

float v  = (w1 + w2 + w3) / 3.0f;  // range [-1, 1]
uint8_t hue = 140 + (uint8_t)(v * 30.0f);
uint8_t bri = 180 + (uint8_t)(v * 70.0f);
leds[xy(x, y)] = CHSV(hue, 230, bri);

Padrão Geométrico

Padrão Geométrico renders rotating concentric rings with a spiral twist. The rings emanate from the centre of the matrix and slowly rotate, giving the impression of a focused, structured energy. It activates on the focused emotion.

☣
Padrão Geométrico
focused — hue 40° (amber)
Ring count proportional to alpha — more rings when relaxed within focus
Rotation speed proportional to beta — spins faster with deeper focus

Core render loop

C++ — pattern_geometric.cpp 
// dx, dy are pixel offsets from centre in [-7.5, 7.5]
float rings = 5.0f + state.alpha * 5.0f;  // 5–10 rings
float rot   = t * 0.002f * (0.5f + state.beta * 2.0f);

float r   = sqrtf(dx*dx + dy*dy);
float ang = atan2f(dy, dx);
float v   = sinf(r * rings - t * 2.0f + ang * 2.0f);

v = v * 0.5f + 0.5f;  // remap to [0, 1]
uint8_t hue = 28 + (uint8_t)(v * 20.0f);   // amber range
leds[xy(x, y)] = CHSV(hue, 240, (uint8_t)(v * 255));

Pulsos Rítmicos

Pulsos Rítmicos combines cross arms radiating from the centre with expanding ring pulses. The cross shape provides a sharp, urgent visual anchor while the rings add a heartbeat-like rhythm. It maps to the high-arousal states: stressed and excited.

⚡
Pulsos Rítmicos
stressed (hue 0° red) · excited (hue 285° magenta)
Pulse rate proportional to beta — formula: 0.8 + beta * 5.0
Arm sharpness Gaussian falloff exp(-dx² × 0.35) — tight beam at centre

Core render loop

C++ — pattern_pulsos.cpp 
float speed = 0.8f + state.beta * 5.0f;

// Cross arms: Gaussian falloff from each axis
float cross = fmaxf(
    expf(-dx*dx * 0.35f),
    expf(-dy*dy * 0.35f)
);

// Expanding ring pulses
float dist  = sqrtf(dx*dx + dy*dy);
float ring  = sinf(dist * 1.6f - t * 0.003f * speed) * 0.5f + 0.5f;

// Blend cross (55%) and ring (45%)
float val   = cross * 0.55f + ring * 0.45f;
leds[xy(x, y)] = CHSV(baseHue, 255, (uint8_t)(val * 255));

Estrelas e Partículas

Estrelas e Partículas fills the matrix with 48 independently twinkling stars. Each star has its own position, twinkle speed, and phase offset, so the field appears organic and alive rather than synchronised. Stars that reach peak brightness also create a subtle 4-pixel cross bloom, simulating the diffraction spikes of a real star. This pattern activates for the neutral emotional state.

✦
Estrelas e Partículas
neutral — hue 200° (blue-magenta)
Star count 48 total; visible density scales with alpha: 10 + alpha × 36
Twinkle speed per-star random speed factor stored in stars[i].speed
Bloom threshold brightness > 180 triggers 4-pixel N/S/E/W neighbours at bri >> 2

Core star loop

C++ — pattern_estrelas.cpp 
uint8_t visible = 10 + (uint8_t)(state.alpha * 36.0f);

for (int i = 0; i < visible; i++) {
  uint8_t bri = sin8(t * stars[i].speed + stars[i].phase);
  leds[xy(stars[i].x, stars[i].y)] = CHSV(stars[i].hue, 200, bri);

  // Bloom: add cross-shaped glow on bright stars
  if (bri > 180) {
    uint8_t bloom = bri >> 2;
    if (stars[i].y > 0)          leds[xy(stars[i].x,   stars[i].y-1)] += CHSV(stars[i].hue, 180, bloom);
    if (stars[i].y < MATRIX_H-1) leds[xy(stars[i].x,   stars[i].y+1)] += CHSV(stars[i].hue, 180, bloom);
    if (stars[i].x > 0)          leds[xy(stars[i].x-1, stars[i].y)]   += CHSV(stars[i].hue, 180, bloom);
    if (stars[i].x < MATRIX_W-1) leds[xy(stars[i].x+1, stars[i].y)]   += CHSV(stars[i].hue, 180, bloom);
  }
}
💡
Tip

The visible star count formula 10 + alpha * 36 means at minimum alpha (0.0) only 10 stars are active — giving a sparse, contemplative look — while at maximum alpha (1.0) all 46 stars light up, creating a dense sparkling field. The remaining 2 stars are kept off as reserve for future use.

Backend Setup

No changes are needed to the Python backend code — the ESP32 uses the same /ws/brain-stream WebSocket endpoint as the browser frontend. The only requirement is that the server binds to all network interfaces so the ESP32 can reach it over WiFi.

Starting the backend for ESP32 access

bash 
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000
⚠
Warning

The default uvicorn command binds to 127.0.0.1 (localhost only). The ESP32 is a separate device on the network — it will never be able to connect unless you explicitly pass --host 0.0.0.0. Verify this every time you start a session.

WebSocket JSON frame structure

Each frame the ESP32 consumes contains the following fields. Only the fields listed here are used by the Arduino sketch — additional fields from the backend are silently ignored.

JSON — EEG data frame 
{
  "alpha":          0.68,
  "beta":           0.18,
  "theta":          0.16,
  "gamma":          0.06,
  "delta":          0.04,
  "emotion":        "calm",
  "confidence":     0.84,
  "signal_quality": 87.3
}

Heartbeat frames

When no EEG session is active, the backend sends periodic heartbeat messages to keep the connection alive. The Arduino sketch detects these before attempting JSON parsing and returns early to avoid spurious errors.

C++ — WebSocket message handler 
// Heartbeat frame looks like: {"type":"heartbeat","status":"waiting"}
void onWebSocketMessage(uint8_t *payload, size_t length) {
  StaticJsonDocument<512> doc;
  deserializeJson(doc, payload, length);

  // Filter heartbeat frames first
  if (doc["type"] == "heartbeat") return;

  // Parse EEG fields
  state.alpha         = doc["alpha"]          | 0.0f;
  state.beta          = doc["beta"]           | 0.0f;
  state.theta         = doc["theta"]          | 0.0f;
  state.emotion       = doc["emotion"]        | "neutral";
  state.signalQuality = doc["signal_quality"]  | 0.0f;
}

Wearable Build

The wearable build focuses on comfort, durability, and long battery life. The LED matrix sits in a sewn pocket on the chest panel of the t-shirt; the ESP32 and power bank live in a small pouch at the hip or in a side pocket. All connections run inside the shirt lining.

Build checklist

  • Use a flexible WS2812B matrix (silicone-encased strip, not a rigid PCB) for comfort against the body and to survive washing.
  • Sew a flat pocket or panel on the t-shirt chest — slightly larger than the matrix — to hold it flat and prevent bunching.
  • Route the data wire and power wires inside the shirt seam from the chest panel down to the hip pouch containing the ESP32.
  • Power the matrix with a 5 V USB power bank. A 10,000 mAh bank lasts 4–6 hours at MAX_BRIGHTNESS = 160.
  • Solder a 330 Ω resistor inline on the data wire between the ESP32 GPIO and the matrix DIN pin.
  • Add a 470 µF capacitor across the 5 V and GND power leads close to the matrix connector to absorb turn-on surge.
  • Use silicone-coated wire for all connections — it remains flexible after many bend cycles unlike PVC hookup wire.
  • Strain-relieve all connectors with a dab of hot glue or a zip tie through a hole in the fabric to avoid broken solder joints during movement.

Power bank selection

Capacity Estimated runtime at brightness 160
5,000 mAh ~2–3 hours
10,000 mAh ~4–6 hours
20,000 mAh ~8–12 hours
💡
Tip — simultaneous charge + discharge

Choose a power bank that explicitly supports pass-through charging (simultaneous charge and discharge). This lets you top up the bank while the shirt is being worn during extended events. Not all power banks support this — check the product specifications before buying.

Troubleshooting

Use the Arduino Serial Monitor at 115200 baud as your first diagnostic tool. Most issues are configuration errors in config.h or power supply problems.

Symptom Likely cause Fix
LEDs don't light at all Wrong GPIO pin Check LED_PIN in config.h. Verify the data wire goes to the correct ESP32 pin and that the matrix GND is connected.
First LED shows wrong colour Wrong COLOR_ORDER Change COLOR_ORDER in config.h — try RGB or BGR. Some WS2812B clones use a different byte order.
Pattern appears upside-down Matrix installed with LED 0 at bottom Toggle MATRIX_FLIP_Y in config.h between true and false.
Pattern mirrors left-right Serpentine wiring direction Toggle MATRIX_SERPENTINE in config.h. Some panels snake in the opposite direction.
No WiFi connection Wrong credentials Double-check WIFI_SSID and WIFI_PASSWORD in config.h. SSID is case-sensitive. Ensure 2.4 GHz band (ESP32 does not support 5 GHz).
WiFi connects but WebSocket never connects Wrong WS_HOST or backend not listening Verify the IP in WS_HOST matches the backend machine. Confirm the backend was started with --host 0.0.0.0, not the default 127.0.0.1.
Pattern never changes No EEG data flowing Start an EEG session in the browser frontend first. The backend only classifies emotions when a BrainFlow session is active.
LEDs flicker or reset Insufficient power supply Use a dedicated 5 V supply rated at least 3 A. Reduce MAX_BRIGHTNESS. Add a bulk capacitor (470–1000 µF) across the power rails near the matrix.
Serial shows JSON parse errors JSON document buffer too small Increase the StaticJsonDocument size in the WebSocket handler — try 1024 bytes instead of 512.
ℹ
Info — watchdog resets

If the ESP32 resets every 5–10 seconds with a Task watchdog got triggered message in the serial monitor, the main loop is blocking for too long — usually caused by a slow WebSocket reconnection attempt. Make sure WS_RECONNECT_MS is at least 3000 ms and that you are not calling delay() with large values inside the render loop.
