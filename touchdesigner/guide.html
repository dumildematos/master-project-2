CHAPTER 01

What is TouchDesigner?

TouchDesigner is a visual programming environment made by Derivative. Instead of writing traditional code line-by-line, you connect nodes (called Operators) on a canvas. Data flows between them in real time — making it ideal for live performances, interactive installations, and in Sentio's case, projecting EEG-driven generative visuals onto physical garments.

The Node-Based Canvas

Every TouchDesigner project lives inside a Network Editor — a zoomable canvas where you place and wire Operators. Think of it like a visual patch bay: each Operator does one job (generate an image, read a sensor, apply a filter) and passes its result to the next node downstream.

💡
Tip

Press Tab on an empty area of the canvas to open the Operator browser. Type the first letters of any Operator to filter the list instantly. Press P to open the parameters panel for the selected node.

Operator Families

Operators are grouped into five families, each with its own wire colour and data type:

TOP
Texture Operator
2D images & video frames. Purple wires. Used for all visual output in Sentio.
CHOP
Channel Operator
Time-series numbers & signals. Green wires. Carries all EEG band values.
DAT
Data Operator
Text, tables & scripts. Teal wires. Python callbacks live here.
SOP
Surface Operator
3D geometry & particles. Blue wires. Used for the spark particle layer.
COMP
Component
Containers, UI & output. Grey wires. Stoner projection mapper lives here.
⚠️
Important

You cannot wire a CHOP directly into a TOP input — they carry different data types. Reference CHOP values inside TOP parameters using the expression syntax op('chop_name')['channel_name'] instead of a physical wire.

ℹ️

The free licence limits output resolution to 1280 × 1280 px. For projecting onto a mannequin in a studio context this is fully sufficient. A paid licence (from $600 USD) removes this restriction.

💡
TouchDesigner 2025 — What's New for Sentio

The latest stable build is 2025.32460 (March 2026). Key changes that affect this project:

  • Python 3.11.10 — all scripts in scripts/ are fully compatible. No changes required.
  • Point Operators (POPs) — a new GPU-based operator family for instanced geometry. Sentio uses the legacy Particle SOP which still works, but POPs are the recommended upgrade path for higher instance counts. See Chapter 5 for details.

  • TDI Library (new) — install the tdi package for full VS Code autocomplete and inline help when editing scripts outside TouchDesigner.
  • Minimum GPU memory: 4 GB — 8 GB recommended if using POPs. Verify your machine meets this before starting.
  • macOS users — macOS 13 Ventura or higher is required for 2025 builds.

CHAPTER 03

Receiving Data via OSC

What is OSC?

OSC (Open Sound Control) is a lightweight network protocol designed for real-time communication between creative software tools. The Sentio Python backend sends EEG band values and emotion data as OSC messages to TouchDesigner over the local loopback network. Each message has a path (like a URL address) and one or more numeric or string values.

OSC message /sentio/alpha → value: 0.72 float 0–1
OSC message /sentio/emotion → value: "calm" string

Adding the OSCin CHOP

Follow these steps inside TouchDesigner to start receiving the Sentio data stream:

1
Open the Operator browser
Press Tab in the empty Network Editor canvas.
2
Add the OSCin CHOP
Type OSC in the search box and double-click OSCin CHOP. A green node appears on the canvas.
3
Open parameters (press P)
With the OSCin node selected, press P to open the Parameters panel on the right side.
4
Set Network Port to 7000
This matches the port configured in the Sentio backend's OSC_PORT environment variable.
5
Set Address Pattern to /sentio
This filters messages — only paths starting with /sentio will be received.
6
Click Active to enable the node
The LED indicator turns green when data is arriving. You should see waveform activity in the node preview.
⚠️
Before you activate

Make sure the Python backend is running first: cd backend && uvicorn main:app --reload Then start a session in the React frontend to trigger the EEG stream. Without an active session the OSCin CHOP will receive no data even if the backend is running.

CHAPTER 04

Building the Processing Network

Raw OSC values are noisy and update at high frequency. Before routing them to the visual operators, build this CHOP processing chain to smooth and scale the signals.

CHOP CHAIN
OSCin CHOP → Filter CHOP → Math CHOP → Null CHOP reference point for TOPs
OSCin CHOP → DAT Execute → Constant CHOP × 3 hue / flow / distortion

Smoothing with Filter CHOP

A Filter CHOP applies a low-pass filter to every incoming channel, removing rapid jitter so visual changes feel fluid rather than twitchy.

ParameterValueWhy
Filter TypeGaussianSmooth bell-curve response, no ringing artefacts
Filter Width0.3 secSmooths ~0.3 s of history; increase for slower visuals
Extend Left / RightHoldPrevents edge artefacts at start/end of stream

Mapping Emotion to Visual Parameters

The emotion label is a string and needs converting to numeric values. Paste this script into a DAT Execute node — it fires every time a new OSC message arrives:

Python — DAT Execute 
# TD 2025 signature — address and args are now passed directly
# Old signature (pre-2022):  onReceiveOSC(dat, message, bytes, timeStamp)
def onReceiveOSC(dat, rowIndex, message, bytes, timeStamp, address, args, peer):
    emotion = str(op('oscin1')['emotion'])

    palettes = {
        'calm':     {'hue': 200, 'flow': 0.15, 'dist': 0.10},
        'focused':  {'hue': 42,  'flow': 0.55, 'dist': 0.30},
        'stressed': {'hue': 0,   'flow': 0.88, 'dist': 0.70},
        'relaxed':  {'hue': 175, 'flow': 0.12, 'dist': 0.06},
        'excited':  {'hue': 285, 'flow': 0.75, 'dist': 0.55},
    }

    p = palettes.get(emotion, palettes['calm'])

    # Write to Constant CHOPs — referenced by TOP parameters
    op('constant_hue').par.value0  = p['hue']
    op('constant_flow').par.value0 = p['flow']
    op('constant_dist').par.value0 = p['dist']
⚠️
TD 2025 — Updated OSC callback signature

TouchDesigner 2022 and later changed the onReceiveOSC callback to include three extra parameters: address, args, and peer. Using the old four-parameter signature still runs without an error, but you cannot access the address or argument values directly — you must read them from the CHOP row instead. The version above uses the correct 2025 signature and gives you all OSC data inline.

CHAPTER 05

Creating the Fluid Generative Visual

This chapter builds the full visual pipeline — an instanced geometry system inspired by Simon Alexander-Adams (Polyhop) in which hundreds of capsule/pill forms drift, rotate and recolour in response to EEG data. The approach mirrors the "Instancing Geometry in TouchDesigner" tutorial aesthetic: dense overlapping translucent shapes noise-driven in position and rotation, with a gradient colour lookup across all instances. Every parameter reacts in real time to incoming EEG data.

TOP CHAIN — instanced geometry pipeline (TD 2025)
Noise TOP → Remap TOP → Blur TOP → Composite TOP → Feedback TOP ↩
Particle SOP → Geometry COMP → Render TOP ↗ merges into Composite (Screen blend)
▲ TD 2025 alternative: POP Network → Render Simple TOP GPU instancing — no Camera/Light needed
Composite → Level TOP → HSV Adjust → Bloom TOP → Out TOP

Noise TOP — the Base Flow Field

The Noise TOP generates a time-varying 2D texture that acts as the flow field driving particle movement and cloth-like deformation.

ParameterValue / ExpressionEffect
TypeSparseOrganic fractal noise — best for fluid look
Roughnessop('filter1')['beta']High beta = chaotic noise structure
Amplitudeop('filter1')['alpha'] * 0.8 + 0.1Alpha controls the noise scale
Period1.5Size of the noise features
Translate ZabsTime.seconds * op('constant_flow').par.value0Animates the noise over time — drives the flow
MonochromeOffAllow colour channels for richer visual

Feedback Loop for Particle Trails

A Feedback TOP blends each new frame with the previous one, creating the long glowing trails characteristic of the Anadol aesthetic.

TouchDesigner — Feedback TOP settings 
# Feedback TOP — wire after the Composite output
Target TOP:   drag your composite output node here
Feedback:     1.0 - op('filter1')['theta'] * 0.3
# High theta → value near 0.7  → shorter trails (meditative state)
# Low theta  → value near 1.0  → very long trails (focused state)

# Then add a Level TOP after Feedback to darken old frames gradually:
Level TOP → Brightness:  0.985   # subtle per-frame fade
💡
Wire order matters

Connect the Feedback TOP output back as the second input of the Composite TOP (not the first). The first input should be the fresh frame from the Noise/Remap chain. If wired backwards the feedback will dominate and the visual will lock up on the first frame.

Instance Layer — Foreground Geometry

The foreground layer replicates the Polyhop instancing aesthetic: hundreds of capsule instances noise-displaced in position and rotation, rendered with additive blending over the noise field. Two approaches are available depending on your TD version and GPU.

💡
Which approach to use in TD 2025?

Particle SOP + Render TOP — works on all TD versions, requires a Camera COMP and Light COMP in the scene.

Point Operators (POPs) + Render Simple TOP — new in TD 2025. GPU-based, no Camera or Light needed. Handles tens of thousands of instances at full frame rate. Use this if your GPU has ≥ 8 GB VRAM.

Option A — Particle SOP (all versions)

1
Add a Particle SOP
Set Birth Rate to op('filter1')['gamma'] * 800 + 100 — gamma bursts create more instances during peak focus.
2
Wire the Noise TOP into Force
Drag the Noise TOP onto the Force input of the Particle SOP. Instances now follow the flow field — they drift in the same noise currents as the background texture.
3
Set Life to 2.0 seconds
Instances persist long enough to form visible overlapping trails before fading.
4
Add Geometry COMP + Render TOP
Wrap the Particle SOP in a Geometry COMP. Add a Camera COMP and Render TOP at the same resolution as your output. Wire the Render TOP into the Composite TOP using Screen blend mode.

Option B — POPs + Render Simple TOP (TD 2025 recommended)

1
Add a POP Network COMP
Press Tab → search POP Network. Inside the network add a Capsule POP as your source geometry and a Noise POP to displace positions and rotations each frame.
2
Drive displacement via EEG
Set the Noise POP's Amplitude to op('/sentio_params')['proc_beta', 1] — beta band drives the turbulence amount, matching the emotion mapping.
3
Add a Render Simple TOP
Press Tab → Render Simple TOP. Set Geometry to your POP Network COMP. No Camera or Light operator needed — the Render Simple TOP handles projection automatically.
4
Wire into Composite TOP as Screen layer
Wire the Render Simple TOP output into the Composite TOP's second input and set blend mode to Screen. The instanced geometry floats above the noise field.

ℹ️
Final Output

Connect everything into a single Out TOP. The Out TOP is what TouchDesigner routes to the Window COMP and ultimately to the projector. All operators upstream feed into it — if the Out TOP is bypassed or missing, nothing appears on the projector.

CHAPTER 06

Projection Mapping onto the Mannequin

Projection mapping makes a flat projector image conform precisely to the surface of a 3D object. In Sentio we project the fluid visual onto a headless dress-form mannequin — a standard dressmaker's torso on a pole stand — turning its white surface into a living EEG canvas that pulses in real time with the wearer's emotional state.

💻
TouchDesigner
sentio.toe
🔌
HDMI Cable
3–5 m length
📽
Projector
1920 × 1080
Mannequin
Dress-form · no head

Physical Setup Checklist

  • Mannequin type: Use a headless torso dress form — no arms, no head, mounted on a centre pole stand. The flat, matte-white surface of a professional dressmaker's form provides the ideal projection canvas. Fabric-padded foam bodies (e.g. standard dressmaking mannequins) work well; avoid hard plastic with a shiny finish.
  • Projector position: Mount directly in front of and slightly above the mannequin. Lens axis should be roughly perpendicular to the torso front face. Ideal throw distance: 1.5 – 2.5 m. Adjust keystone in the projector's on-screen menu if the image tapers at top or bottom.
  • Room lighting: Darken the room as much as possible. Ambient light washes out projected colour. Black-out curtains or a dark exhibition booth work best.
  • Mannequin surface: White or very light grey matte surfaces reflect projected colour most faithfully. Avoid glossy or sequined fabric — they create hotspot reflections and colour shifts.
  • Stand stability: The mannequin must not move during or after calibration. Clamp the pole base to the floor with gaffer tape or use a heavy tripod-style weighted base. Even 2–3 cm of movement will misalign the projection.
  • Projection area: Frame the projector output to cover from the shoulder line to the hip/waist hem of the dress form — roughly 40–60 cm tall on the physical surface. Crop using the Stoner COMP control points (see Step 5 below) rather than changing the projector zoom.
  • Projector resolution: Set to the projector's native resolution (e.g. 1920 × 1080). Match this value in the TouchDesigner Window COMP settings.
  • Laptop placement: Keep the laptop running Sentio within HDMI cable reach. A 3–5 m cable avoids clutter near the projection area.

Stoner COMP — the Projection Mapping Tool

Stoner is TouchDesigner's built-in projection mapping component. It lets you define an arbitrary polygon matching the dress form's torso outline and warps the input texture to fill that shape precisely.

1
Add a Stoner COMP
Press Tab → search Stoner → double-click to place it.
2
Wire your Out TOP into Stoner's Texture input
Drag the wire from your Out TOP node to the left input of the Stoner COMP.
3
Add a Window COMP and wire Stoner into it
The Window COMP opens the fullscreen output window on the projector display.
4
In the Window COMP: set Display to your projector
Set Display to the projector's display index (usually Display 2). Enable Fullscreen and click Open as Separate Window.
5
Double-click Stoner to enter its canvas
You'll see a quad overlay on a grey background representing the projection area.
6
Click Edit Points in the Stoner controls
Corner handles appear. You are now ready for calibration.

Calibrating Corner-Pins

Corner-pin calibration aligns the projected image with the physical dress form surface. This is the most critical step for a convincing result — take your time.

1
Use a white test texture first
Temporarily wire a white Constant TOP into Stoner. A solid white projection lets you see the boundary on the dress form clearly.
2
Project an alignment grid
Replace the Constant TOP with a Grid TOP — the grid lines make distortion immediately visible so you can judge accuracy.
3
Drag the four corner handles
In Stoner's Edit Points mode, drag each corner handle to match the dress form's outer torso boundary — from neck/shoulder down to hip hem. Work clockwise from top-left.
4
Add mesh points for curved edges
Click Add Point in Stoner and place extra control points along the shoulder slope and hip flare to follow the 3D curvature of the dress form.
5
Swap in the fluid visual
Once the grid aligns cleanly, rewire the fluid Out TOP back into Stoner. The visual now conforms to the dress form shape.
6
Save immediately — Ctrl + S
All Stoner calibration data is stored inside the .toe file. Next time you open the project the calibration is restored automatically.
🚨
Do not move the dress form after calibration

Even a few centimetres of movement will misalign the projection. If the dress form or its pole stand shifts accidentally, re-run the calibration steps from Step 3. Before starting, mark the base of the pole stand on the floor with gaffer tape so it can be repositioned precisely if needed.

CHAPTER 09

Connecting Operators to Python Scripts

This chapter shows exactly how to wire each operator in the TouchDesigner network to its Python script. Follow these steps after opening a blank project — or run build_network.py from the Textport to do it all automatically.

Automatic Build (Recommended)

The fastest way to wire everything is to let the builder script do it:

1
Open TouchDesigner with a blank project
File → New, or just launch TD — the default empty project is fine.
2
Open the Textport — Alt + T
The Textport is TouchDesigner's Python console. It lives at Dialogs → Textport & DATs.
3
Open build_network.py in any text editor
It lives at touchdesigner/build_network.py. Select all and copy.
4
Paste into the Textport and press Enter
The script prints a live build log as it creates and wires every operator. Takes about 3 seconds.
5
Save — Ctrl + S → name it sentio.toe
Save inside the touchdesigner/ folder so the script paths resolve correctly.
💡
Script path auto-detection

build_network.py resolves the scripts/ folder relative to the saved .toe file path. Save the project before running the builder if you want scripts to load automatically — otherwise set SCRIPTS_DIR at the top of the file to the absolute path.

Manual Wiring — DAT Operators

If you prefer to build manually, follow these wiring instructions for each operator type.

sentio_params
Table DAT
no script — data store
The shared memory of the entire network. All scripts read from and write to this table.
  • Add DAT → Table DAT. Name it exactly sentio_params.
  • No script attached. Rows are pre-populated by startup_init.py on load.
  • Column 0 = key name, Column 1 = value (string or float).
startup_exec
Execute DAT
startup_init.py
Runs once when the project loads. Populates sentio_params defaults and verifies all operators exist.
  • Add DAT → Execute DAT. Name it startup_exec.
  • In Parameters: set Start = On, Frame Start = Off, all others Off.
  • Paste the contents of startup_init.py into the DAT text area.
  • No input wire needed — it uses op() lookups internally.
execDat
Execute DAT
execute_dat.py
The main frame loop. Calls all module scripts in the correct order every frame.
  • Add DAT → Execute DAT. Name it execDat.
  • In Parameters: set Frame Start = On, Start = Off, all others Off.
  • Paste the contents of execute_dat.py into the text area.
  • No input wire — it loads each module via op('mod_*').module.
osc_dat_exec
DAT Execute
osc_callbacks.py
Fires on every incoming OSC message. The single entry point for all EEG data.
  • Add DAT → DAT Execute DAT. Name it osc_dat_exec.
  • Wire oscin1 → osc_dat_exec input connector.

Frame Execution Order

The execDat Execute DAT calls every module script in this exact order on each frame. The order matters — each script depends on data written by the script before it.

1
signal_processor.py
onFrameStart(frame)
Reads raw bands from sentio_params → applies smoothing → writes proc_* values and pushes to constant_flow / constant_dist CHOPs.
2
signal_quality_monitor.py
onFrameStart(frame)
Evaluates rolling quality window → classifies state → enters or exits fallback mode → writes mode to sentio_params.
3
emotion_mapper.py
tick_transition(frame)
Advances the emotion preset blend → writes all preset_* rows to sentio_params → updates constant_hue CHOP.
4
color_palette_manager.py
tick_palette_transition(frame)
Advances palette blend → extracts dominant hue → updates hsvAdjust1.par.hueshift and hsvAdjust1.par.satmult.
5
param_handler.py
onFrameStart(frame)
Reads preset_* + proc_* rows → applies per-band nonlinear curves → smooth-writes to noise1, blur1, feedback1, level1, hsvAdjust1, bloom1.
6
particle_controller.py
onFrameStart(frame)
Reads bands + preset_particle_rate → maps to birth rate, lifespan, turbulence, drag → writes to particles1, geo1, render1.
7
debug_utils.py (optional)
tick_watches(frame)
Prints any registered operator parameter watches to the Textport. Only active when watches are registered manually.
ℹ️
OSC callbacks run separately

osc_callbacks.py runs outside this frame loop — it fires as a DAT Execute callback whenever a new OSC packet arrives from the backend (up to ~60 times per second). It writes raw values to sentio_params which the frame loop then picks up on the next frame.
